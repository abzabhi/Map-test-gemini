Genesis
Thought experiment. Vibe coded using Claude AI (almost) fully. Started with a simple query — optimise my golf bag for a course — and here we are. Philosophically 100% free. I hope this helps my game...

GORDy stands for Golf Optimization Research & Decisions — with a lowercase y added to make it sound like an actual caddie.
About This App
Virtual Caddie is a personal bag management and course strategy tool. All Optimizations are done by Claude AI or some other AI and can vary based on model and model provider. Use at your own risk. All in browser calculations are deterministic — the same inputs always produce the same outputs. This tab documents some of the key assumptions baked into those calculations.
Handicap & Scoring
WHS Differential
Formula: (Gross Score − Course Rating) × 113 ÷ Slope Rating, rounded to one decimal. This matches the World Handicap System standard.
Handicap Index
Uses best differentials from round history per the WHS table: 1 round → best 1; 7–8 rounds → best 2; 9 rounds → best 3; 10–11 → best 4; 12–14 → best 5; 15–16 → best 6; 17–18 → best 7; 19–20 → best 8. Average of selected differentials × 0.96 (WHS playing conditions calculation factor). All rounds are stored; only the most recent 20 eligible rounds are used for handicap calculation.
Playing Handicap
Formula: round(HCP Index × (Slope ÷ 113) + (Course Rating − Par)). Applies to the selected tee and course.
Stroke Allocation
Holes are sorted by their stroke index (handicap column, 1 = hardest). A player receives 1 stroke on any hole whose index ≤ Playing HCP. If Playing HCP > 18, a second stroke is allocated on holes whose index ≤ (Playing HCP − 18).
Club Distance & Implied Yardage
Session Averages
avgMin and avgMax are simple means of all logged session values for that club. Each session contributes equally regardless of date.
Yardage Type — Carry vs Total
Each club can be logged as Carry (ball flight only) or Total (carry + roll). Set a global default in Profile, or override per club in its specs panel. When the Viz tab is set to Total and a club is logged as Carry, the app converts automatically using the roll % table. The reverse applies when Viz is set to Carry.
Carry to Total Conversion (Roll %)
Roll is expressed as a percentage of carry distance and varies by club type and handicap tier. Higher-lofted clubs land steeper and roll less; lower-lofted clubs run out more. Example: a Driver at tier 3 (HCP 15–19) rolls approximately 16% of carry — a 230 yd carry becomes ~267 yds total. Wedges at tier 5+ roll 0–1% (high spin, holds or spins back). Values are based on TrackMan, Arccos, and MyGolfSpy data under typical firm-to-moderate fairway conditions. Soft/wet conditions reduce roll by 3–5%; firm/links conditions add 5–8%.
Implied Yardage — How It Works
Implied yardage adjusts only the minimum (floor) of the range based on your confidence score. The maximum always stays at avgMax — the app assumes your best strike is your ceiling, and confidence reflects how reliably you reach the floor.
  • Confidence 5 — floor raised by 5% (consistent striker, tight window)
  • Confidence 4 — no adjustment, raw avg floor used (baseline)
  • Confidence 3 — floor lowered by 5% (some variability)
  • Confidence 1–2 — floor lowered by 10% (high variability, treat range as wider)
Gap suppression: if avgMax − avgMin is already tight (≤ 15 yds for short clubs; ≤ 20 yds for long clubs), confidence scaling is skipped entirely — the raw range is already precise enough and further adjustment would distort it. Toggle implied yardage display on/off in the Clubs tab.
Long vs Short Club Threshold
Driver, Fairway Wood, hybrids ≤ 6H or loft ≤ 30° → Long (20 yd gap threshold). Hybrids ≥ 7H or loft > 30°, Irons, Wedges, Chipper → Short (15 yd gap threshold).
Simple vs Detailed Club Entry
The Clubs tab has a Simple mode (default) and a Detailed mode toggle. Simple mode asks only for max distance and derives the minimum using a 20-yard default window. Confidence is set to 3 automatically in Simple mode. Detailed mode lets you enter both min and max explicitly and set all specs. Use Simple for new clubs and switch to Detailed once you have real session data.
Optimisation Tiers
When you export a session file, you choose an optimisation tier. This tells the AI how much depth to apply. The tier is embedded in the export and the AI reads it on load.
Tier 1 — Basic
Midpoint yardage only: round((impMin + impMax) / 2). No session data required. Output is club name and midpoint. Fastest; suitable when you have minimal data or just want a rough bag list.
Tier 2 — Standard
Full implied ranges (impMin–impMax), gap and overlap analysis, stroke allocation per hole. Brief reasoning per club. Adds YARDAGE GAPS and basic COURSE STRATEGY. Best for everyday use.
Tier 3 — Advanced
Everything in Tier 2, plus dispersion radius, bias offset, longitudinal miss probability for forced carries, hole notes from your data, handedness adjustment. Pulls in prior session and round history at the same course. Recommended when you have multiple sessions of data.
Tier 4 — Full
Everything in Tier 3, plus conditions woven throughout, confidence scaling rationale in LEFT OUT notes, fitting insights, full strategy with stroke allocation called out hole by hole. Most thorough; longest output. Use when preparing for a meaningful round at a known course.
What the AI Optimises For
At all tiers: only clubs marked Active with at least one distance session are eligible. Putter is always carried and never counted in the 14-club limit. Clubs are selected to maximise coverage with minimal gaps, weighted by the holes actually in play (front 9, back 9, or all 18). At Tier 2+, the AI factors in stroke allocation — holes where you get extra strokes lower the bar for a bogey and change which distances matter most.
Ball Flight & Dispersion
Sources
Spin RPM and launch angle benchmarks from TrackMan published amateur data. Lateral dispersion calibrated against GOLFTEC launch monitor study and DECADE / Practical Golf. Longitudinal miss distances from HackMotion research. Roll percentages from Arccos GPS analytics and MyGolfSpy ball speed studies. Arccos GPS data used for cross-validation.
Handicap Tiers
Six tiers used for dispersion and roll lookups: 0–4 · 5–9 · 10–14 · 15–19 · 20–24 · 25+. If no handicap is set, tier 3 (15–19) is assumed.
Iron Classification
Irons are split into Long (2–4i, loft ≤ 27°), Mid (5–7i, 28–38°), and Short (8–9i, 39–46°) groups for dispersion lookup. Classification uses the loft° value first, then the iron number.
Lateral Dispersion
Expressed as a ± radius (half of total lateral width). Adjusted by bias multiplier: Straight ×1.00 · Draw ×0.95 · Fade ×0.95 · Hook ×1.15 · Slice ×1.20 · Push Right ×1.00 · Push Left ×1.00. Lateral offset applied to centreline: Draw −5 yds · Hook −15 yds · Fade +5 yds · Slice +15 yds · Push Right +20 yds · Push Left −20 yds (positive = right for right-handed; mirrored for left-handed). Push biases translate the entire pattern sideways rather than widening it.
Longitudinal Dispersion
Short and long miss distances are tracked separately — players at all levels miss shorter more than longer. These inform forced-carry decisions in the caddie session.
Ellipse Asymmetry
The dispersion ellipse is not a symmetric oval. Right misses (open face) produce higher spin and shorter carry — the ellipse is narrower on the right. Left misses (closed face / hook) travel farther, making the ellipse wider on the left. The ellipse tilts clockwise from vertical for right-handed players. Push biases have near-symmetric ellipses since the miss is directional, not spin-induced.
Visualisation
Bag Coverage Mode
Shows stacked dispersion ellipses for the selected club set, sorted by carry distance descending. Gap markers appear between clubs where the gap exceeds 12 yards. Fairway hit % chips are colour-coded: green ≥ 60%, amber 35–59%, red < 35%. Clubs can be sourced from an AI optimisation session (auto-preselects the recommended bag) or a custom selection of up to 13 clubs (excluding Putter).
Hole Planning Mode
Renders a single hole as a fairway corridor scaled to the hole's yardage from the selected tee. Overlays tee shot and approach dispersion ellipses at their correct positions from the tee marker. Club selection defaults to the AI caddie session's hole-by-hole recommendation and updates automatically as you move between holes. The advisory note from the session is shown above the canvas for context.
Carry / Total Toggle
Switches all ellipse widths and distance markers between carry and total yardage. Total is the default. When a club is logged as Carry and Total is selected, distances are scaled up using the roll % table. When logged as Total and Carry is selected, distances are scaled down.
Display Modes
Both — shape outline plus probability shading (default). Shape — outline only, no fill. Prob — probability zones only, no outline. Four shaded zones per ellipse represent right miss, left miss, short miss, and long miss probabilities derived from the bias and longitudinal tables.
Fairway Width
Entered in yards. Used to calculate the fairway hit percentage shown in the chips below the canvas. Adjust to match the hole you are planning for.
AI Caddie Sessions
How Sessions Work
Export a session file from the GORDy tab and upload it to any AI chat (Claude, ChatGPT, etc.). The AI reads your clubs, course, and handicap, then produces a bag optimisation and hole-by-hole strategy. Import the result back into the app via the Caddie Session card in GORDy.
Structured Data Blocks
When the AI produces a session result, it appends two machine-readable blocks that the app uses for visualisation: a BAG-MAP listing the exact recommended club identifiers in order, and a HOLE-MAP listing the primary tee and approach club for each hole. These are stripped from the displayed scorecard text — they exist solely for the app to drive the Viz tab without text parsing.
AI Output Varies
Recommendations depend on the AI model and provider in use. The same input may produce different advice on different models or on different days. Treat all AI output as a starting point, not a definitive prescription.
Home & Favourite Course
Home course is set manually by toggling the button on any course card. Only one course can be home at a time. The home course always appears first in the course list and is shown in your profile tagline.

Favourite course is calculated automatically as the course name appearing most frequently in your round history. It updates in real time as you log rounds. If the favourite is the same as home, only the Home badge is shown.
Data & Privacy
All personal data is stored exclusively in your browser's localStorage. Nothing is transmitted to any server operated by this app. Use Save Data to download a backup file — this is the only way to transfer data between devices or browsers. Clearing browser data will erase all app data; always save first.

Cross-device sync (optional): if configured, your data is encrypted in-browser using AES-GCM before transmission and stored on Cloudflare's infrastructure under your chosen sync ID and passphrase. Anthropic has no access to your data. The skill and reference files the app fetches on first load are hosted on GitHub Gist (read-only, no personal data).

If you export a session file to an AI service, that data is processed under the terms of that AI provider.
Reference Data & Versions
The app fetches skill and reference data from GitHub Gist on load and caches it locally. The reference data includes dispersion tables, carry-to-total roll percentages, confidence scaling rules, bias multipliers, and yardage defaults by club type and handicap tier. When the reference version is updated, the cache refreshes automatically on next load.

The FlightDataVersion field in exported data files records which version of the dispersion engine was active when the session was run. This allows the AI caddie to detect mismatches between stored sessions and the current tables.
⚠️ Clear All Data
This will permanently erase all clubs, courses, rounds, history, and profile data from this device.

Have you saved your data file? Use Save Data first to download a backup — once cleared, this action cannot be undone.

If you have a sync profile connected, your encrypted data remains on the server and is not affected — you can pull it again after clearing. If you rely only on local files, make sure you have downloaded a backup before proceeding.
This action is permanent and irrevocable.
